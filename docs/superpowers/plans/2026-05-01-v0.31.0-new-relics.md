### Task 1: 扩展 RelicData 枚举 + RelicEngine 处理分支

**Files:**
- Modify: `RelicData.gd` line 5
- Modify: `RelicEngine.gd` lines 14–19

#### 当前代码

`RelicData.gd` 第 1–12 行：
```gdscript
class_name RelicData
extends Resource

enum Trigger { COMBAT_START, TURN_START }
enum EffectType { ENERGY, HEAL_HP }

@export var display_name: String = ""
@export var description: String = ""
@export var trigger: Trigger = Trigger.COMBAT_START
@export var effect_type: EffectType = EffectType.ENERGY
@export var value: int = 0
@export var price: int = 0
```

`RelicEngine.gd` 第 14–19 行（`_apply` 函数）：
```gdscript
static func _apply(relic: RelicData, engine: CombatEngine) -> void:
	match relic.effect_type:
		RelicData.EffectType.ENERGY:
			engine.energy += relic.value
		RelicData.EffectType.HEAL_HP:
			engine.player.hp = mini(engine.player.hp + relic.value, engine.player.max_hp)
```

- [ ] **Step 1: 修改 RelicData.gd 枚举**

将第 5 行改为：

```gdscript
enum EffectType { ENERGY, HEAL_HP, BLOCK, DRAW }
```

- [ ] **Step 2: 修改 RelicEngine.gd _apply()**

将 `_apply()` 函数改为：

```gdscript
static func _apply(relic: RelicData, engine: CombatEngine) -> void:
	match relic.effect_type:
		RelicData.EffectType.ENERGY:
			engine.energy += relic.value
		RelicData.EffectType.HEAL_HP:
			engine.player.hp = mini(engine.player.hp + relic.value, engine.player.max_hp)
		RelicData.EffectType.BLOCK:
			engine.player.add_block(relic.value)
		RelicData.EffectType.DRAW:
			engine._draw_cards(relic.value)
```

说明：
- `add_block()` 是 `Combatant` 的现有公共方法
- `_draw_cards()` 是 `CombatEngine` 的现有方法（GDScript 中下划线前缀仅为命名约定，可从 RelicEngine 调用）

- [ ] **Step 3: 提交**

```bash
git add RelicData.gd RelicEngine.gd
git commit -m "feat: add BLOCK and DRAW effect types to relic system"
```

参考格式来自同目录 `life_ring.tres`（无 UID，直接用 path 引用脚本）。
