# v0.32.0 宝箱节点实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在地图第 6 列（column index 5）新增固定宝箱节点，玩家点击后随机获得一个 CHEST 遗物，通过复用 RewardScreen 展示（隐藏卡牌选择区域）。

**Architecture:** 扩展 RelicData 增加 source 字段区分遗物来源；NodeConfig 新增 CHEST 类型；MapGenerator 将 column 5 固定为 [CHEST, REST]；GameManager 新增 go_to_chest() 并用 pending_card_reward 字段控制 RewardScreen 是否显示卡牌选项。

**Tech Stack:** Godot 4 GDScript，Resource .tres 数据文件

	if col <= 3:

- [ ] **Step 2: 修改 MapGenerator.gd，将 col <= 5 拆分为 col == 4 和 col == 5**

将 `_get_column_types` 函数改为：

```gdscript
static func _get_column_types(col: int) -> Array:
	if col <= 3:
		return [NodeConfig.Type.COMBAT, NodeConfig.Type.COMBAT]
	elif col == 4:
		var types: Array = [NodeConfig.Type.COMBAT, NodeConfig.Type.REST]
		types.shuffle()
		return types
	elif col == 5:
		return [NodeConfig.Type.CHEST, NodeConfig.Type.REST]
	elif col <= 7:
		var types: Array = [NodeConfig.Type.ELITE, NodeConfig.Type.COMBAT]
		types.shuffle()
		return types
	elif col == 8:
		return [NodeConfig.Type.ELITE, NodeConfig.Type.ELITE]
	else:
		push_error("unexpected column %d" % col)
		return [NodeConfig.Type.COMBAT, NodeConfig.Type.COMBAT]
```

说明：
- column 5 固定返回 `[CHEST, REST]`（row 0 = 宝箱，row 1 = 休息），不洗牌
- `_add_shop_nodes` 只替换 `type == COMBAT` 的节点，CHEST 节点不受影响，无需修改
