# v0.33.0 职业选择 + 剑修身份设计

**目标：** 在主菜单和游戏开始之间插入职业选择界面，显示三个职业（法修/体修暂时锁定），选择剑修后开始游戏；剑修携带本命法宝养剑葫（暂无剑意效果，v0.34.0 实现）。

**架构：** 新建 CharSelectScreen 场景，对应新 GameManager Phase；PlayerState 记录职业；CardPool 管理职业起始遗物路径；养剑葫通过新增 EffectType.SWORD_INTENT 占位，暂不触发任何效果。

---

## 数据层

### RelicData.gd

`EffectType` 枚举新增 `SWORD_INTENT = 4`：

```gdscript
enum EffectType { ENERGY, HEAL_HP, BLOCK, DRAW, SWORD_INTENT }
```

RelicEngine 的 match 无对应分支，SWORD_INTENT 暂时静默（无效果），v0.34.0 补实现。

## 逻辑层

### GameManager.gd

`Phase` 枚举新增 `CHAR_SELECT`：

```gdscript
enum Phase { MENU, CHAR_SELECT, MAP, COMBAT, REWARD, REST, SHOP, WIN, GAME_OVER }
```

新增方法 `go_to_char_select()`：

```gdscript
func go_to_char_select() -> void:
    current_phase = Phase.CHAR_SELECT
    get_tree().change_scene_to_file("res://char_select/CharSelectScreen.tscn")
```

`start_new_run()` 新增 `character: String` 参数，根据职业加载起始遗物：

```gdscript
func start_new_run(character: String) -> void:
    pending_card_reward = true
    player_state = PlayerState.new()
    player_state.character = character
    # 加载起始遗物
    var relic_paths: Array[String] = []
    match character:
        "sword":
            relic_paths.assign(CardPool.SWORD_START_RELICS)
    for path in relic_paths:
        player_state.relics.append((load(path) as RelicData).duplicate())
    # 以下与原版相同（起始牌组、地图生成等）
    ...
```

### menu/MainMenu.gd

"开始游戏"按钮改为调用 `go_to_char_select()`：

```gdscript
func _on_start_pressed() -> void:
    GameManager.go_to_char_select()
```

## 游戏流程

1. 主菜单点击"开始游戏" → CharSelectScreen
2. 点击"剑修" → `start_new_run("sword")`
3. 剑修携带养剑葫进入游戏（显示在战斗界面遗物栏）
4. 养剑葫暂无效果（v0.34.0 实现剑意时补上）
