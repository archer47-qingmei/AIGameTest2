# 游戏元素详情面板 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 点击遗物或 buff/debuff 标签，弹出居中详情面板，显示名称和描述。

**Architecture:** 新建 `GameText.gd` 静态类存储 buff 描述；新建 `InfoPanel.tscn / InfoPanel.gd` 通用弹窗组件；修改 `CombatScreen.gd` 将遗物 Label 和 buff Label 改为 Button，点击时调用 InfoPanel。

**Tech Stack:** GDScript 4, Godot 4.4, Control/Panel UI 节点

---

## 触发与交互

- **触发方式**：点击（鼠标左键 / 触屏点击）
- **覆盖元素**：战斗界面中的所有遗物标签、虚弱/脆弱/力量/剑意 buff 标签
- **关闭方式**：点击半透明遮罩背景 或 点击「关闭」按钮

## InfoPanel 结构

```
全屏 ColorRect 遮罩（Color(0,0,0,0.5)，鼠标事件拦截）
└─ 居中 Panel（宽 260px，高度由内容撑开，不设固定高）
   ├─ Label：标题（字号 20，加粗）
   ├─ HSeparator
   ├─ RichTextLabel：描述（自动换行，滚动禁用）
   └─ Button：「关闭」
```

- InfoPanel 初始 `visible = false`
- `show_info(title: String, description: String)` 设置文本并显示
- `hide_info()` 隐藏面板
- InfoPanel 是 CombatScreen 的子节点，`z_index = 10`

## GameText 静态类

文件：`GameText.gd`（`class_name GameText`，`extends Object`）

```gdscript
static func get_buff_info(key: String) -> Dictionary:
    # 返回 {name: String, description: String}
```

覆盖以下 key：

| key | 名称 | 描述 |
|---|---|---|
| `weak` | 虚弱 | 造成的伤害降低 25%，每回合结束减少 1 层。 |
| `vulnerable` | 脆弱 | 受到的伤害增加 25%，每回合结束减少 1 层。 |
| `strength` | 力量 | 每点力量使普通攻击伤害 +1。 |
| `sword_intent` | 剑意 | 积累剑意可提升下次攻击伤害，每层 +1 伤害，上限 10 层。 |

未知 key 返回 `{name: key, description: "（暂无说明）"}`。

## CombatScreen 改动

### 遗物列表（_refresh_ui 中）

原来：为每个 `RelicData` 创建 `Label`。  
改为：创建 `Button`，`flat = true`（无边框样式），`.text = r.display_name`，`.pressed` 连接到调用 `_info_panel.show_info(r.display_name, r.description)`。

### Buff 标签（_build_status_row 中）

原来：为虚弱/脆弱创建 `Label`。  
改为：创建 `Button`，`flat = true`，`.text = "虚弱×N"` / `"脆弱×N"`，`.pressed` 连接到调用 `_info_panel.show_info(...)` 并从 `GameText.get_buff_info("weak"/"vulnerable")` 取描述。

### InfoPanel 引入

- 在 `CombatScreen.tscn` 中添加 `InfoPanel` 子场景节点
- `@onready var _info_panel: InfoPanel = $InfoPanel`
