# v0.42.0 休息站二选一 + 升级预览 设计文档

**目标：** 休息站改为二选一（休息 OR 升级卡牌），升级时支持点击卡牌预览升级属性（数值有提升则显绿色）。

**架构：** 改动集中在 `rest/RestScreen.gd`、`rest/RestScreen.tscn`、`CardData.gd`、`GameManager.gd`，不涉及 CombatEngine 等战斗逻辑。

**Tech Stack：** Godot 4 GDScript，RichTextLabel BBCode，Resource 数据层。

---

## 文件结构

| 文件 | 动作 | 职责 |
|------|------|------|
| `GameManager.gd` | 修改 | `go_to_rest()` 移除 `apply_rest_heal()` 调用 |
| `PlayerState.gd` | 不改 | `apply_rest_heal()` 保留，由 RestScreen 主动调用 |
| `CardData.gd` | 修改 | 新增 `get_upgrade_preview_bbcode() -> String` |
| `rest/RestScreen.gd` | 重写 | 两阶段逻辑：选择阶段 → 结果阶段 |
| `rest/RestScreen.tscn` | 重写 | 新增选择面板、预览面板 |

## 功能 1：二选一逻辑

### 当前行为

`GameManager.go_to_rest()` 在切换场景前直接调用 `apply_rest_heal()`，玩家到达休息站时已经被动回血；界面同时显示回血结果和卡牌升级列表。

### 新行为

**阶段一（选择）：** 进入休息站后显示两个按钮：
- 「休息（+X 生命）」——X 为 `int(max_hp * REST_HEAL_RATIO)`，只预览不实际回血
- 「升级卡牌」

**阶段二A（选了休息）：**
- 调用 `GameManager.player_state.apply_rest_heal()`
- 显示"恢复了 X 点生命 / 当前生命：HP / MaxHP"
- 显示「继续」按钮

**阶段二B（选了升级）：**
- 显示可滚动卡牌列表（已升级的卡禁用）
- 点击卡牌 → 弹出预览面板（见功能 2）
- 升级确认后禁用所有卡牌，显示「继续」按钮

### 实现要点

- `GameManager.go_to_rest()` 删除 `player_state.apply_rest_heal()` 一行
- RestScreen 在 `_ready()` 只显示选择面板；`LblHeal`、`LblHP`、卡牌列表、预览面板默认隐藏

## 功能 2：升级预览（点击查看 + 绿色高亮）

### 交互流程

1. 玩家在卡牌列表中点击一张牌 → 预览面板出现
2. 预览面板显示：
   - 当前描述（普通颜色）
   - 升级后描述（有变化的数值用绿色高亮）
3. 点「确认升级」→ 执行 `card.upgrade()`，关闭预览，禁用所有卡牌，显示「继续」
4. 点「返回」→ 关闭预览，回到卡牌列表

### `get_upgrade_preview_bbcode() -> String`

在 `CardData.gd` 中新增此方法：**不修改卡牌数据**，仅根据当前 `effect.value` 和 `effect.upgrade_bonus` 计算升级后的数值，生成带 BBCode 颜色标签的描述字符串。

格式：
```
[b]斩击+[/b]
费用:1  攻:[color=green]9[/color]  挡:3
```

- 若某字段 `upgrade_bonus > 0`，该数值用 `[color=green]值[/color]` 包裹
- 若所有 `upgrade_bonus == 0`（如纯名字升级），文字正常颜色，追加说明"（属性不变）"

生成方式：复用 `get_description()` 的数值提取逻辑，对有 upgrade_bonus 的字段额外用绿色标签包裹，最终拼成 BBCode 字符串。

### 预览面板节点（tscn）

```
PanelPreview (Panel)
  └─ VBoxContainer
       ├─ LblCurrent (Label)        — 当前描述（普通文本）
       ├─ RtlUpgraded (RichTextLabel) — 升级后描述（BBCode）
       ├─ HBoxContainer
       │    ├─ BtnConfirmUpgrade (Button) — "确认升级"
       │    └─ BtnCancelPreview (Button)  — "返回"
```

## 不在本版范围

- 卡牌升级的视觉动画
- 升级后的持久化（已有：`card.is_upgraded = true`，重新进入战斗生效）
- 多张牌升级（每次休息只能升级一张）

- [ ] 选"升级卡牌" → 显示卡牌列表，不触发回血

- [ ] 有数值提升的字段显示绿色；无提升则显示普通颜色并标注"（属性不变）"
- [ ] 点「确认升级」→ 升级生效，列表禁用，出现「继续」
- [ ] 点「返回」→ 关闭预览，可再选另一张牌
- [ ] 版本号显示 v0.42.0
