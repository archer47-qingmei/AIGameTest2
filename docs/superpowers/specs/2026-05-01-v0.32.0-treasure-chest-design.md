# v0.32.0 宝箱节点设计

**目标：** 在地图第 6 列（column index 5）新增固定宝箱节点；玩家点击后随机获得一个来源为 CHEST 的遗物，复用现有 RewardScreen 展示，无需新建场景。

**架构：** 扩展 RelicData（新增 source 字段）、NodeConfig（新增 CHEST 类型）、MapGenerator（固定列生成宝箱节点）、GameManager（新增 go_to_chest() 和 pending_card_reward 控制字段）、RewardScreen（条件性隐藏卡牌选择区域）。

---

## 数据层

### RelicData.gd

新增 `source` 枚举字段，标记遗物来源：

```gdscript
enum Source { SHOP, CHEST }
@export var source: Source = Source.SHOP
```

### 遗物来源标记（修改 .tres）

| 文件 | source 值 | 说明 |
|------|-----------|------|
| iron_armor.tres | 1（CHEST） | 进入宝箱池 |
| tome_of_wisdom.tres | 1（CHEST） | 进入宝箱池 |
| burning_gem.tres | 0（SHOP，默认） | 不变，为战斗奖励 |
| life_ring.tres | 0（SHOP，默认） | 不变，为战斗奖励 |

## 地图层

### NodeConfig.gd

`Type` 枚举新增 `CHEST` 值：

```gdscript
enum Type { COMBAT, REST, ELITE, SHOP, CHEST }
```

### MapGenerator.gd

- 列 5（column index 5）改为固定生成 `[CHEST, REST]`，取代原来的 `[COMBAT, REST]`
- `_add_shop_nodes()` 的随机商店替换范围不得包含 CHEST 节点（保持宝箱固定位置）

---

## 逻辑层

### GameManager.gd

新增字段：

```gdscript
var pending_card_reward: bool = true
```

新增方法 `go_to_chest()`：

```gdscript
func go_to_chest() -> void:
    var chest_relics: Array[RelicData] = [
        preload("res://data/relics/iron_armor.tres"),
        preload("res://data/relics/tome_of_wisdom.tres"),
    ]
    pending_relic = chest_relics.pick_random().duplicate()
    pending_gold = 0
    pending_card_reward = false
    go_to_reward()
```

`select_node()` 新增分支：

```gdscript
NodeConfig.Type.CHEST: go_to_chest()
```

`go_to_map()` 重置 `pending_card_reward = true`（确保后续奖励界面正常显示卡牌选择）。

## UI 层

### RewardScreen.gd

`_show_list()` 中，在创建"选择卡牌"按钮前加条件判断：

```gdscript
if GameManager.pending_card_reward:
    # 创建选择卡牌按钮
```

其余逻辑（遗物展示、金币、继续按钮）完全复用，无需改动。

### MapScreen.gd

CHEST 节点按钮文字显示为 `"宝箱"`，样式与商店节点相同（使用相同样式分支）。

---

## 游戏流程

1. 玩家在地图第 6 列看到"宝箱"节点
2. 点击宝箱 → `select_node()` → `go_to_chest()`
3. 随机选取 CHEST 池遗物，设置 `pending_relic`，`pending_card_reward = false`
4. 加载 RewardScreen：只显示遗物奖励，无金币、无卡牌选择
5. 玩家点击领取遗物 → 点击继续 → 返回地图
6. `go_to_map()` 重置 `pending_card_reward = true`

---

## 不在本版范围内

- 宝箱节点专属动画或开箱特效
- 重复遗物去重逻辑（同一遗物可重复持有）
- 宝箱列位置随机化
- 宝箱遗物池动态扫描（目前用固定 preload 列表）
