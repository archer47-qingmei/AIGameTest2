# v0.9.0 HP 跨关卡持续设计文档

**日期**：2026-04-28  
**项目**：AIGameTest2（Godot 4.6 + GDScript）  
**范围**：v0.9.0 — 玩家 HP 在同一局内跨关卡持续，两场战斗之间无回血

---

## 背景

v0.8.0 建立了 2 节点线性地图，但 `CombatEngine.setup()` 每次都将玩家 HP 硬编码重置为 70，导致奖励选牌之外的资源（HP 损耗）对游戏进程毫无影响。v0.9.0 让 HP 真正成为跨关卡持续的资源，使两场战斗的策略张力成立。

---

## 架构原则遵守情况

- **PlayerState**：数据层，新增 `hp` 字段，仍为纯 RefCounted 数据容器
- **CombatEngine**：逻辑层，setup() 接受外部 `initial_hp` 参数，不再硬编码，与 v0.8.0 接受 `enemy_data` 的模式一致
- **CombatScreen**：UI 层，负责在胜利时将战斗结果（final HP）写回 `GameManager.player_state.hp`，与 RewardScreen 写回 `player_state.deck` 的既有模式一致
- **GameManager**：不变，`start_new_run()` 无需修改（PlayerState 默认值已处理初始化）

---

## 文件结构

```
res://

└── combat/
```

## 模块设计

### PlayerState.gd（修改）

```gdscript
class_name PlayerState
extends RefCounted

var deck: Array[CardData] = []
var current_node: int = 0
var hp: int = 70
```

`hp` 默认值 70，`PlayerState.new()` 时自动初始化，`GameManager.start_new_run()` 无需额外赋值。败北时 `go_to_menu()` 将 `player_state` 置 null，hp 随之销毁。

---

### CombatEngine.gd（修改）

`setup()` 新增 `initial_hp: int` 参数，移除硬编码的 `player.hp = 70`：

```gdscript
func setup(initial_deck: Array[CardData], enemy_data: EnemyData, initial_hp: int) -> void:
    _enemy_data = enemy_data
    enemy = Combatant.new()
    enemy.display_name = _enemy_data.display_name
    enemy.hp = _enemy_data.hp
    enemy.max_hp = _enemy_data.hp
    enemy.block = 0

    player = Combatant.new()
    player.display_name = "玩家"
    player.hp = initial_hp
    player.max_hp = 70
    player.block = 0

    for card: CardData in initial_deck:
        _draw_pile.append(card.duplicate())
    _draw_pile.shuffle()
    _start_player_turn()
```

`player.max_hp` 保持硬编码 70，v0.9.0 不引入最大 HP 变化机制。

---

### combat/CombatScreen.gd（修改）

两处改动：

**`_ready()`**：setup 调用新增第三参数：

```gdscript
_engine.setup(GameManager.player_state.deck, GameManager.get_current_enemy_data(), GameManager.player_state.hp)
```

**`_on_combat_ended()`**：胜利分支写回 HP，然后再做按钮显示判断：

```gdscript
func _on_combat_ended(result: String) -> void:
    _lbl_result.text = "胜利！" if result == "victory" else "游戏结束"
    _lbl_result.visible = true
    _btn_end_turn.disabled = true
    if result == "victory":
        GameManager.player_state.hp = _engine.player.hp
        if GameManager.is_final_node():
            _btn_win.visible = true
        else:
            _btn_get_reward.visible = true
    else:
        _btn_return_menu.visible = true
    for btn: Button in _hand_buttons:
        btn.disabled = true
```

败北时不写回 HP（`player_state` 将在 `go_to_menu()` 中置 null）。

## 游戏流程

```
start_new_run()
  PlayerState.hp = 70（默认值）
  └─→ MapScreen → 关卡一战斗
        CombatEngine.setup(..., initial_hp=70)
        ├─ 胜利（假设剩余 HP=45）
        │     player_state.hp = 45
        │     └─→ RewardScreen → 选牌 → MapScreen
        │           └─→ 关卡二战斗
        │                 CombatEngine.setup(..., initial_hp=45)
        │                 ├─ 胜利 → player_state.hp = final_hp → WinScreen
        │                 └─ 败北 → go_to_menu() → player_state = null
        └─ 败北 → go_to_menu() → player_state = null
```

## v0.9.0 验收标准

- [ ] 新局开始时玩家 HP 为 70
- [ ] 关卡一战斗受伤（HP < 70）后胜利，进入奖励界面
- [ ] 奖励选牌后进入地图，再进入关卡二时 HP 显示为关卡一结束时的值（不是 70）
- [ ] 关卡一满血通过时，关卡二以 HP=70 开始
- [ ] 任意关卡败北返回主菜单，再开新局 HP 重置为 70
- [ ] 全部 v0.8.0 验收标准继续通过
