# v0.12.0 查看牌组 设计文档

**日期：** 2026-04-28
**目标：** 战斗中新增"查看牌组"弹窗，玩家可通过 3 个 Tab 分别查看完整牌组、抽牌堆、弃牌堆。

---

## 核心决策

| 问题 | 决策 |
|---|---|
| Tab 数量 | 3 个：完整牌组 / 抽牌堆 / 弃牌堆 |
| 卡牌显示内容 | 卡名 + 费用 + 效果描述（复用 `CardData.get_description()`） |
| 数据访问方式 | CombatEngine 新增 getter 方法，保持封装 |
| 弹窗位置 | 内联在 CombatScreen.tscn，不新建独立场景 |

---

## 架构总览

```
逻辑层
  CombatEngine.gd    新增 get_draw_pile() / get_discard_pile() 两个公开方法

UI 层
  combat/CombatScreen.tscn   新增隐藏 DeckViewPanel（Panel + TabContainer）
  combat/CombatScreen.gd     新增"查看牌组"按钮逻辑
```

**数据流：**

```
玩家点击"查看牌组"
  → CombatScreen 读取 engine.hand / engine.get_draw_pile() / engine.get_discard_pile()
  → 清空并填充 3 个 Tab 的 VBoxContainer（每张卡一个 Label）
  → 显示 DeckViewPanel
  → 玩家点击"关闭" → DeckViewPanel.hide()
```

完整牌组 Tab = hand + draw_pile + discard_pile 合并（不排序）。

---

## 逻辑层（CombatEngine.gd）

新增两个公开 getter 方法（其余代码不动）：

```gdscript
func get_draw_pile() -> Array[CardData]:
    return _draw_pile

func get_discard_pile() -> Array[CardData]:
    return _discard_pile
```

`hand` 已是公开变量，无需新增 getter。返回直接引用，UI 层只读使用。

---

## UI 层

### CombatScreen.tscn 新增节点

```
DeckViewPanel (Panel, visible=false)
  └── VBoxContainer
        ├── TabContainer
        │     ├── ScrollContainer (name="完整牌组")
        │     │     └── VBoxContainer [_all_cards_list]
        │     ├── ScrollContainer (name="抽牌堆")
        │     │     └── VBoxContainer [_draw_list]
        │     └── ScrollContainer (name="弃牌堆")
        │           └── VBoxContainer [_discard_list]
        └── Button ("关闭") → DeckViewPanel.hide()
```

### CombatScreen.gd 新增逻辑

- `@onready` 引用 `_all_cards_list`、`_draw_list`、`_discard_list`、`_deck_view_panel`
- 新增"查看牌组"Button，连接 `_on_view_deck_pressed()`
- `_on_view_deck_pressed()`：
  1. 清空 3 个 VBoxContainer 的子节点
  2. 填充完整牌组：遍历 `engine.hand + engine.get_draw_pile() + engine.get_discard_pile()`
  3. 填充抽牌堆：遍历 `engine.get_draw_pile()`
  4. 填充弃牌堆：遍历 `engine.get_discard_pile()`
  5. 每张卡创建 `Label`，`text = card.get_description()`
  6. 调用 `_deck_view_panel.show()`

**弹窗不阻断游戏流程**：Panel 可视时遮挡底层按钮，玩家无法误触出牌；关闭后自动恢复，无需额外状态管理。

## 验收标准

- [ ] 战斗界面有"查看牌组"按钮
- [ ] 点击后弹出含 3 个 Tab 的面板

- [ ] "弃牌堆" Tab 只显示当前弃牌堆中的牌
- [ ] 每张卡显示完整描述（复用 get_description()）
- [ ] 点击"关闭"弹窗消失，游戏正常继续
