# v0.28.0 毒液牌机制与新敌人 Design Spec

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 新增毒液牌机制——敌人可将毒液牌塞入玩家抽牌堆，回合结束时手牌中留存的毒液牌每张扣 1 HP；同时新增两种使用该机制的中后期敌人。

**Architecture:** 纯扩展，不改变现有接口。`CardData` 新增 `is_venom` 标识字段；`CombatEngine` 扩展 `end_turn()` 结算逻辑和 `_do_enemy_turn()` 的 poison 分支；新增两个敌人 `.tres` 和一个毒液卡 `.tres`；`MapGenerator` 拆分敌人池支持按列范围分配。

**Tech Stack:** Godot 4 GDScript，数据层 `.tres` 文件。

---

## Section 1: 毒液牌机制

### CardData.gd

新增一个导出字段（必须 `@export` 才能在 `.tres` 中序列化）：

```gdscript
@export var is_venom: bool = false
```

`get_description()` 末尾特判：

```gdscript
if is_venom:
    return "毒液\n费用:0  留手扣血"
```

### data/cards/venom.tres

新建卡牌资源：
- `card_name = "毒液"`
- `cost = 0`
- `is_venom = true`
- `effects = []`（无效果，打出仅将牌移到弃牌堆）

### CombatEngine.gd — end_turn() 扩展

在弃牌前统计手牌中的毒液牌数量并直接扣 HP（绕过 block），然后检查死亡：

```gdscript
func end_turn() -> void:
    var venom_count: int = 0
    for card: CardData in hand:
        if card.is_venom:
            venom_count += 1
    if venom_count > 0:
        player.hp = max(0, player.hp - venom_count)
        if _check_end():
            return
    for card: CardData in hand:
        _discard_pile.append(card)
    hand.clear()
    _do_enemy_turn()
    state_changed.emit()
    if not _check_end():
        _start_player_turn()
```

### CombatEngine.gd — _do_enemy_turn() 扩展

在现有 `if action.type == "attack"` 分支后新增 `elif action.type == "poison"` 分支：

```gdscript
elif action.type == "poison":
    var venom_card: CardData = load("res://data/cards/venom.tres") as CardData
    for i in action.value:
        _draw_pile.append(venom_card.duplicate())
    _draw_pile.shuffle()
```

### 战斗结束清除

毒液牌仅存在于 `CombatEngine` 内部的 `_draw_pile` / `_discard_pile` / `hand` 中，这些结构在战斗结束后随 engine 实例一起丢弃，**无需额外清理**——不会写入 `PlayerState.deck`。

## Section 3: MapGenerator 分池

### 新增常量

```gdscript
const EARLY_ENEMY_PATHS: Array[String] = [
    "res://data/enemies/jaw_worm.tres",
    "res://data/enemies/fire_lizard.tres",
]
const MID_ENEMY_PATHS: Array[String] = [
    "res://data/enemies/poison_spider.tres",
    "res://data/enemies/swamp_slime.tres",
]
```

删除原 `COMBAT_ENEMY_PATHS` 常量。

### _random_combat_enemy 接收 col 参数

```gdscript
static func _random_combat_enemy(col: int) -> EnemyData:
    var paths: Array[String] = EARLY_ENEMY_PATHS if col <= 3 else MID_ENEMY_PATHS
    return load(paths[randi() % paths.size()]) as EnemyData
```

### _make_column 传入 col

在 `_make_column()` 中对 COMBAT 节点调用改为：

```gdscript
NodeConfig.Type.COMBAT:
    nd.config.enemy_data = _random_combat_enemy(col)
```
