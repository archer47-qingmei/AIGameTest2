# 杀戮尖塔风格随机地图 设计文档

## 目标

将现有全连通固定列地图改为路径优先的随机地图，每列最多 3 个节点，连线只沿生成路径绘制，视觉效果接近《杀戮尖塔》。

## 架构

MapGenerator 完全重写为路径优先算法；NodeConfig 新增 `offset` 字段存储随机偏移；MapScreen 的坐标计算改用偏移字段；GameManager / PlayerState 接口不变。

**技术栈：** GDScript 4，Godot 4.4

---

## 算法：路径生成

### 参数
- 路径数：5 条
- 列数：10（列 0-8 为普通关卡，列 9 为 Boss）
- 每列行数：0-2（共 3 行）

### 步骤

1. 每条路径随机选起始行（0/1/2）
2. 每一列转移时，路径随机选择：上移一行 / 保持 / 下移一行，结果 clamp 到 [0, 2]
3. 同一 `(列, 行)` 位置只创建一个 NodeData（路径合并）
4. 记录连线：路径从 `(列 N, 行 A)` 走到 `(列 N+1, 行 B)`，则在 node_A.connections 中加入 node_B（去重）
5. 最后一步：列 8 的所有节点强制连向唯一 Boss 节点（列 9，行 1）

### 结果

- 每列实际节点数 1-3，全图约 15-18 个节点
- 连线稀疏，有明显路径感

## 节点布局

### 画布

- 宽：600px（原 480px）
- 高：1650px（不变）

### 坐标

基准坐标按行分配，生成时加一次随机偏移并存入 NodeConfig.offset：

| 行 | 基准 X | 随机偏移 |
|----|--------|---------|
| 0  | 100    | ±25px   |
| 1  | 300    | ±25px   |
| 2  | 500    | ±25px   |

Y 坐标 = 1500 − (列 × 150)，随机偏移 ±20px。

Boss 节点固定在 (300, 80)，无偏移。

偏移在 MapGenerator 生成时计算，存入 NodeConfig.offset，MapScreen 读取时直接使用，保证多次进入地图界面坐标不变。

---

## 节点类型分配

路径生成完成后，按列分配类型：

| 列  | 规则 |
|-----|------|
| 0-3 | 全部 COMBAT |
| 4   | 节点数 ≥2：随机 1 个改为 REST；否则保持 COMBAT |
| 5   | 节点数 ≥2：1 个改为 CHEST，1 个改为 REST，其余保持 COMBAT；只有 1 个则 CHEST |
| 6-7 | 节点数 ≥2：随机 1 个改为 ELITE；其余 COMBAT |
| 8   | 全部 ELITE |
| 9   | 单一 BOSS 节点 |

Shop 注入：列 4-7 中随机将 2 个 COMBAT 节点改为 SHOP（现有逻辑保留）。

## 改动范围

### MapGenerator.gd（完全重写）
- 移除 `_make_column()` 和全连通逻辑
- 新增路径生成算法（5 条路径，随机游走，合并同位置节点）
- 新增坐标偏移计算，写入 NodeConfig.offset
- 保留节点类型分配和 Shop 注入逻辑（按新规则调整）

### NodeConfig.gd（新增字段）
- `@export var offset: Vector2 = Vector2.ZERO`

### MapScreen.gd（坐标计算）
- `_get_node_pos(nd)` 改为：基准坐标（按行）+ nd.config.offset
- 画布宽度常量从 480 改为 600
- ScrollContainer / MapContent 最小宽度对应更新

### GameManager.gd — 不改动
### PlayerState.gd — 不改动
