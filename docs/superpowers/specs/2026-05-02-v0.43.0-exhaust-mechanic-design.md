# v0.43.0 功法 Exhaust 机制 设计文档

**目标：** 功法类卡牌打出后进入消耗区，不进入弃牌堆，不会被重新洗入抽牌堆。

**架构：** 改动集中在 `CombatEngine.gd`（逻辑层）和 `CombatScreen.gd` / `CombatScreen.tscn`（UI 层），不涉及 CardData、EffectResolver 等其他模块。

**Tech Stack：** Godot 4 GDScript，TabContainer。

---

## 功能 1：消耗逻辑（CombatEngine）

### 当前行为

`play_card()` 最后一步：`_discard_pile.append(card)`，所有打出的牌都进弃牌堆，功法牌会被重新洗入抽牌堆可再次使用。

### 新行为

`play_card()` 按 `card.card_type` 分流：
- `card_type == "功法"` → `_exhaust_pile.append(card)`
- 其他 → `_discard_pile.append(card)`（原有逻辑不变）

新增：
- `var _exhaust_pile: Array[CardData] = []`
- `func get_exhaust_pile() -> Array[CardData]` — 返回 `_exhaust_pile.duplicate()`

### 关于效果叠加

`_apply_engine_effects()` 现有实现已是加法逻辑（`sword_intent_cap += effect.value` 等）。若玩家持有多张相同功法牌，分别打出后效果自然累积，无需修改。

## 功能 2：消耗区 UI（CombatScreen）

### 场景变更（tscn）

在 `DeckViewPanel/VBoxContainer/TabContainer` 下新增第四个标签节点：

```
[node name="消耗区" type="ScrollContainer" parent="DeckViewPanel/VBoxContainer/TabContainer"]
  └─ ExhaustList (VBoxContainer)
```

### 脚本变更（gd）

新增 `@onready` 引用：
```gdscript
@onready var _exhaust_list: VBoxContainer = $DeckViewPanel/VBoxContainer/TabContainer/消耗区/ExhaustList
```

`_on_view_deck_pressed()` 新增一行：
```gdscript
_populate_list(_exhaust_list, _engine.get_exhaust_pile())
```

## 不在本版范围

- 消耗区专用图标或视觉样式
- 通过卡牌效果将消耗区的牌取回（不实现）
- 招式/身法牌的 exhaust 机制（只有功法牌 exhaust）
