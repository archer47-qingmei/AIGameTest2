# 机缘事件系统 Design Spec

**版本：** v0.69.0
**日期：** 2026-05-04

---

## 目标

在地图上新增「机缘」节点类型，玩家进入后触发随机叙事事件，通过选项获得奖励或承受代价。本版本搭建完整系统框架并实装 12 件事件验证流程。

---

## 架构概览

```
数据层（Resource / .tres）
  EventData.gd          ← 单个事件蓝图
  EventChoiceData.gd    ← 单个选项蓝图
  EventEffectData.gd    ← 单条效果蓝图
  data/events/*.tres    ← 12件事件数据文件

逻辑层（RefCounted）
  EventEngine.gd        ← 纯静态效果执行器
  EventPool.gd          ← 事件路径注册表（类似 CardPool）

地图接入
  NodeConfig.gd         ← 新增 EVENT 节点类型
  MapGenerator.gd       ← 生成 EVENT 节点（每局 2-3 个）
  GameManager.gd        ← go_to_event() + select_node 分支

UI 层（Control）
  event/EventScreen.tscn / .gd  ← 事件主界面（含内嵌卡牌选择面板）

其他改动
  PlayerState.gd        ← 新增 reincarnation_fragments 字段
  rest/RestScreen.gd    ← UI 文本「升级」→「悟道」
```

## 数据层

### EventData.gd

```gdscript
class_name EventData
extends Resource

@export var display_name: String = ""
@export var mood: String = ""                    # "安静" / "惊悚" / "神秘" / "有趣" / "感动"
@export var flavor_text: String = ""             # 场景描述文案
@export var choices: Array[EventChoiceData] = []
@export var appear_condition: AppearCondition = AppearCondition.NONE
@export var appear_condition_value: int = 0      # 条件数值（境界号 / 牌数 / 遗物数 / HP%）
@export var character_requirement: String = ""   # "" = 通用，"sword"/"fa"/"ti" = 角色专属

enum AppearCondition {
    NONE,
    AFTER_REALM,            # 通过第 N 境界后（value = 境界号）
    MIN_DECK_SIZE,          # 牌组 >= N 张
    MIN_RELIC_COUNT,        # 拥有 >= N 件遗物
    MIN_HP_PERCENT,         # 当前 HP >= N%
    MIN_UNUPGRADED_ATTACKS  # 牌组中 >= N 张未悟道招式
}
```

### EventChoiceData.gd

```gdscript
class_name EventChoiceData
extends Resource

@export var button_text: String = ""    # 选项按钮文字，如"深入开采"
@export var result_text: String = ""    # 选择后显示的叙述文案
@export var effects: Array[EventEffectData] = []
@export var cost_gold: int = 0          # >0 表示需要花费灵石才能选择
```

### EventEffectData.gd

```gdscript
class_name EventEffectData
extends Resource

enum EffectType {
    NOTHING,
    GAIN_GOLD,
    LOSE_GOLD,
    GAIN_HP,
    LOSE_HP,
    HEAL_PERCENT_MAX,
    HEAL_FULL,
    LOSE_HP_PERCENT_CURRENT,
    LOSE_MAX_HP_PERCENT,
    GAIN_RELIC,
    LOSE_RELIC_RANDOM,
    GAIN_CARD,
    GAIN_CURSE_CARD,
    REMOVE_CARD_CHOOSE,
    REMOVE_CARDS_TYPE,
    REMOVE_CARDS_RANDOM,
    UPGRADE_CARD_CHOOSE,
    UPGRADE_CARDS_TYPE,
    COPY_CARD_CHOOSE,
    GAIN_REINCARNATION_FRAGMENT,
    GAIN_MAX_HP_PERCENT,
    GAIN_ENERGY_CAP,
    COMBAT_ELITE_STUB
}

@export var effect_type: EffectType = EffectType.NOTHING
@export var value: int = 0
@export var probability: float = 1.0    # 1.0 = 必触发，0.7 = 70% 概率
@export var card_grade: int = 0         # 0=任意, 1=凡品, 2=灵品, 3=仙品
@export var relic_grade: int = 0        # 0=任意, 1=法器, 2=灵器, 3=仙器
@export var card_type_filter: String = ""  # "招式"/"身法"/"心魔"/"走火入魔" 等
```

```gdscript
class_name EventPool
extends RefCounted

const EVENTS: Array[String] = [

    "res://data/events/lao_xiu_shi_yan_dou.tres",
    "res://data/events/ku_jing.tres",
    "res://data/events/si_ceng_xiang_shi.tres",
    "res://data/events/ya_bi_ke_zi.tres",
    "res://data/events/jian_hen.tres",

# 从池中随机选一个满足出现条件的事件
static func pick_event(player_state: PlayerState) -> EventData
```

### EventEngine.gd

```gdscript
class_name EventEngine
extends RefCounted

# 检查事件是否满足出现条件
static func check_condition(event: EventData, player_state: PlayerState) -> bool

# 检查选项是否可选（灵石是否足够）
static func check_choice_available(choice: EventChoiceData, player_state: PlayerState) -> bool

# 执行一个选项的所有立即效果（非交互类）
# 返回需要交互的效果列表（UPGRADE_CARD_CHOOSE / REMOVE_CARD_CHOOSE / COPY_CARD_CHOOSE）
static func apply_immediate_effects(choice: EventChoiceData, player_state: PlayerState) -> Array[EventEffectData]

# 执行单条交互效果（选牌完成后调用）
static func apply_interactive_effect(effect: EventEffectData, selected_card: CardData, player_state: PlayerState) -> void

# 生成效果的中文摘要文本（用于 UI 显示）
static func describe_effects(choice: EventChoiceData) -> String
```

**概率效果处理**：`probability < 1.0` 时，`_apply_effect` 内部用 `randf()` 判断是否触发，不暴露给调用方。

**COMBAT_ELITE_STUB**：显示提示文字「[此功能将在后续版本实装]」，不触发战斗。

### GameManager.gd 改动

`select_node()` 新增分支：
```gdscript
NodeConfig.Type.EVENT: go_to_event()
```

新增方法：
```gdscript
func go_to_event() -> void:
    get_tree().change_scene_to_file("res://event/EventScreen.tscn")
```

### MapGenerator.gd 改动

在第 1-3 列和第 5 列（现有 CHEST_REST 列）中，将部分 COMBAT 节点替换为 EVENT。目标每局生成 2-3 个 EVENT 节点，通过概率控制（约 20% 的非保留位替换为 EVENT）。

### NodeConfig.gd 改动

```gdscript
enum Type { COMBAT, REST, ELITE, SHOP, CHEST, BOSS, EVENT }
```

### PlayerState.gd 改动

```gdscript
var reincarnation_fragments: int = 0
```

## UI 层

### EventScreen 流程

**阶段一：主界面**
- 显示：色调标签、事件名称、场景描述文案、选项按钮列表
- 灵石不足的选项按钮置灰
- 点击可用选项 → 进入阶段二

**阶段二：结果视图**（隐藏选项区，显示结果区）
- 显示：选项结果文案、效果摘要文字（如「获得 30 灵石」）
- 「确认」按钮
- 点击确认：
  - 若效果含交互类（UPGRADE_CARD_CHOOSE / REMOVE_CARD_CHOOSE / COPY_CARD_CHOOSE）→ 显示卡牌选择面板（阶段三）
  - 否则 → 执行所有效果，调用 `GameManager.go_to_map()`

**阶段三：卡牌选择面板**（叠加在主界面上）
- 显示提示文字（「请选择一张牌悟道」等）
- 滚动列表展示可选卡牌（按 card_type_filter 过滤）
- 已满足条件的牌高亮可选，不满足的置灰
- 「确认」按钮（选牌后才可点）
- 完成后执行交互效果，调用 `GameManager.go_to_map()`

### RestScreen 改动

将「升级」文案替换为「悟道」，仅涉及 UI 文本，逻辑不变。

## 推迟到后续版本

- **交错类**法修/体修专属事件：数据文件写好，`character_requirement` 设置正确，但不加入 `EventPool.EVENTS`
- **COMBAT_ELITE_STUB** 实装：后续版本将 stub 替换为真实精英战斗跳转
- **复杂事件**：雾中墟市（多选一 UI）、镜湖幻境/藏经阁残卷（复制牌）、散功重修/无尽雷池/天地熔炉（复杂牌组操作）

---

## 不在本版本做的事

- 轮回碎片实际效果（元剧情推进）：本版本只计数
- 临时 buff（「下 3 场战斗多抽 1 张牌」）：需要跨战斗状态系统，后续版本实现
- 法修 / 体修角色
