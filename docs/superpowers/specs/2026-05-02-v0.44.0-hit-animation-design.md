# v0.44.0 受击动画 + 伤害数字飞出 设计文档

**目标：** 敌人被攻击时播放受击动画（抖动 + 红色闪烁），同时显示伤害数字并执行飞出动画（放大后随机向左或向右飘移淡出）。

**架构：** `CombatEngine` 新增 `damage_dealt` 信号传递伤害量；`CombatScreen` 监听信号，用 Tween 驱动受击与数字飞出动画。不修改任何 `.tscn` 文件，Label 节点运行时动态创建。

**Tech Stack：** Godot 4 GDScript，Tween API。

## 功能 1：damage_dealt 信号（CombatEngine）

### 新增信号

```gdscript
signal damage_dealt(enemy_index: int, amount: int)
```

### emit 位置

在 `play_card()` 内，调用 EffectResolver 前后对比每个敌人的 HP，对 HP 有减少的敌人 emit 信号。

实现方式：在 `_apply_card_effects()` 调用前，记录所有敌人的 HP 快照；调用后比较差值，对每个 `hp_before - hp_after > 0` 的敌人 emit `damage_dealt(index, hp_before - hp_after)`。

> 注：`state_changed` 信号不受影响，仍在原位置 emit。

## 功能 2：受击动画（CombatScreen）

### 连接信号

在 `_ready()` 中连接 `_engine.damage_dealt.connect(_on_damage_dealt)`。

### `_on_damage_dealt(enemy_index: int, amount: int)`

通过 `enemy_index` 找到对应的 Button 节点（敌人 Button 列表按索引对应）。

对该 Button 执行两组并行 Tween：

**抖动（shake）**
- 用 `Tween.tween_property` 修改 Button 的 `position.x`（相对偏移）
- 序列：+8px → -8px → +5px → -5px → 0px，每步约 0.05s，共 0.25s
- 动画结束后恢复原始 position

**红色闪烁（flash）**
- 用 `Tween.tween_property` 修改 Button 的 `modulate`
- 序列：`Color(1, 0.2, 0.2)` → `Color(1, 1, 1)`，时长 0.25s

### 数字飞出

动态创建 `Label` 节点，添加为 CombatScreen 的直接子节点（避免被 EnemiesContainer 裁剪）。

初始状态：
- `text = str(amount)`
- `position` = 目标 Button 的 `global_position` + Button 尺寸的中心偏移，转换为 CombatScreen 本地坐标
- `scale = Vector2(1.0, 1.0)`
- `modulate.a = 1.0`

Tween 序列（全程约 0.4s）：
1. **放大**：`scale` 从 `Vector2(1, 1)` → `Vector2(1.5, 1.5)`，时长 0.15s，缓动 `EASE_OUT`
2. **飘移 + 淡出**（与放大并行）：`position.x` 随机 ±60px（`randf_range(-60, -20)` 或 `randf_range(20, 60)`，两侧随机），时长 0.4s；`modulate.a` 从 1.0 → 0.0，时长 0.4s
3. Tween 完成后 `queue_free()`

## 不在本版范围

- 玩家受击动画（本版只处理敌人）
- 敌人攻击时的攻击动画
- 治疗数字（不同颜色）
- 暴击或特殊效果的差异化表现
