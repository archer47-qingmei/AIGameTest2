# v0.10.0 抽牌与能量卡牌设计文档

**日期**：2026-04-28  
**项目**：AIGameTest2（Godot 4.6 + GDScript）  
**范围**：v0.10.0 — 新增 draw / energy 两种引擎级效果类型，新增 4 张卡牌扩充奖励池

---

## 背景

v0.9.0 完成 HP 跨关卡持续，玩家在两场战斗中需要管理资源。当前奖励池只有 4 种卡牌（打击/防御/重击/斩击），全部是 damage/block 效果，选择缺乏策略深度。v0.10.0 引入"抽牌"和"能量回复"两种新效果类型，并新增 4 张卡牌，使奖励选择真正具备差异化的策略权衡。

---

## 架构原则遵守情况

- **EffectResolver**：接口完全不变（仍只处理 Combatant 层的 damage/block），不引入新依赖
- **CombatEngine**：新增私有方法 `_apply_engine_effects()` 和 `_draw_cards()`，专门处理引擎层效果（draw/energy），与 EffectResolver 职责明确分离
- **CardData**：`get_description()` 扩展以显示 draw/energy，纯逻辑改动
- **数据层**：4 张新卡均为 `.tres` 文件，RewardEngine 自动枚举 `data/cards/` 下全部文件，无需任何代码改动

## 模块设计

### CombatEngine.gd（修改）

在 `play_card()` 中，`EffectResolver.resolve()` 调用之后插入 `_apply_engine_effects(card)`：

```gdscript
func play_card(card: CardData) -> void:
    if energy < card.cost:
        return
    energy -= card.cost
    EffectResolver.resolve(card, player, enemy)
    _apply_engine_effects(card)
    var idx: int = hand.find(card)
    if idx >= 0:
        hand.remove_at(idx)
    _discard_pile.append(card)
    state_changed.emit()
    _check_end()

func _apply_engine_effects(card: CardData) -> void:
    for effect: CardEffectData in card.effects:
        if effect.type == "draw":
            _draw_cards(effect.value)
        elif effect.type == "energy":
            energy += effect.value

func _draw_cards(n: int) -> void:
    for i: int in n:
        if _draw_pile.is_empty():
            if _discard_pile.is_empty():
                break
            for card: CardData in _discard_pile:
                _draw_pile.append(card)
            _discard_pile.clear()
            _draw_pile.shuffle()
        if _draw_pile.is_empty():
            break
        var card: CardData = _draw_pile.pop_back()
        hand.append(card)
```

`_draw_hand()` 保持不变，`_draw_cards()` 向现有手牌追加卡牌。

### CardData.gd（修改）

`get_description()` 改为只显示非零字段，并支持 draw/energy：

```gdscript
func get_description() -> String:
    var dmg: int = 0
    var blk: int = 0
    var drw: int = 0
    var nrg: int = 0
    for effect: CardEffectData in effects:
        if effect.type == "damage":   dmg = effect.value
        elif effect.type == "block":  blk = effect.value
        elif effect.type == "draw":   drw = effect.value
        elif effect.type == "energy": nrg = effect.value
    var desc: String = "%s\n费用:%d" % [card_name, cost]
    if dmg > 0: desc += "  攻:%d" % dmg
    if blk > 0: desc += "  挡:%d" % blk
    if drw > 0: desc += "  抽:%d" % drw
    if nrg > 0: desc += "  能:%d" % nrg
    return desc
```

## 游戏效果说明

| 卡牌 | 费用 | 效果 | 策略定位 |
|------|------|------|---------|
| 洞察 | 1 | 抽 2 张牌 | 加速牌组循环，配合高价值卡使用 |
| 迅刺 | 0 | 攻击 4 点，抽 1 张牌 | 免费输出 + 补牌，连击启动器 |
| 蓄力 | 1 | 获得 3 能量（净 +2） | 为大费用卡蓄能，高费牌组支撑 |
| 冲刺 | 0 | 攻击 4 点，获得 1 能量 | 免费输出 + 补能，灵活填充 |

这 4 张卡自动进入 RewardEngine 奖励池（枚举 `data/cards/*.tres`），无需改动 RewardEngine。
