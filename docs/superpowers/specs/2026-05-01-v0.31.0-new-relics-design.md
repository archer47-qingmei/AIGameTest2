# v0.31.0 新遗物「铁皮护甲」「智者之书」设计

**目标：** 新增两个遗物，覆盖格挡和手牌两个新维度，使遗物体系更完整。

**架构：** 在 `RelicData.gd` 枚举中新增 `BLOCK` 和 `DRAW` 效果类型；`RelicEngine._apply()` 新增对应处理分支；新建两个 `.tres` 数据文件。UI 层无需改动。

---

## 新遗物一览

| | 铁皮护甲 | 智者之书 |
|--|----------|----------|
| 触发 | 每回合开始（TURN_START） | 每回合开始（TURN_START） |
| 效果 | 获得 4 点格挡 | 多抽 1 张牌 |
| 售价 | 120 金 | 130 金 |

与现有遗物合计覆盖四个维度：能量（燃烧宝石）、回血（生命之环）、格挡（铁皮护甲）、手牌（智者之书）。

## 数据层

### RelicData.gd

`EffectType` 枚举新增两个值：

```gdscript
enum EffectType {
    ENERGY,   # 已有：增加能量
    HEAL_HP,  # 已有：恢复生命
    BLOCK,    # 新增：获得格挡
    DRAW,     # 新增：抽牌
}
```

### data/relics/iron_armor.tres（新建）

```
display_name = "铁皮护甲"

trigger      = 1   # TURN_START

```

## 逻辑层（RelicEngine.gd）

`_apply()` 函数在现有两个 match 分支之后新增：

```gdscript
RelicData.EffectType.BLOCK:
    engine.player.add_block(relic.value)

RelicData.EffectType.DRAW:
    engine._draw_cards(relic.value)
```

- `add_block()` 是 `Combatant` 的现有公共方法
- `_draw_cards()` 是 `CombatEngine` 的现有方法，GDScript 中下划线前缀仅为命名约定，RelicEngine 可以调用
