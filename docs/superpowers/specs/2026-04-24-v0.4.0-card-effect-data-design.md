# v0.4.0 CardEffectData 设计文档

**日期**：2026-04-24  
**项目**：AIGameTest2（Godot 4.6 + GDScript）  
**范围**：v0.4.0 — 引入 `CardEffectData` 替换 `CardData` 的 `damage`/`block` 硬编码字段（纯重构，零功能变化）

---

## 背景

v0.3.0 将卡牌数据外置为 `.tres` 文件，但 `CardData` 仍有固定的 `damage` 和 `block` 两个字段，每新增一种效果类型（如抽牌、虚弱、中毒）都需要修改 `CardData.gd` 和 `EffectResolver.gd`。

v0.4.0 目标：将效果列表抽象为 `Array[CardEffectData]`，为多效果卡牌打好基础，本版本游戏行为与 v0.3.0 完全一致。

---

## 新增 Resource 类

### CardEffectData.gd

```gdscript
class_name CardEffectData
extends Resource

@export var type: String   # "damage" 或 "block"
@export var value: int
```

与 `EnemyActionData` 完全对称，保持代码风格一致。

**职责边界：**
- `CardEffectData` = 静态效果蓝图（存文件，只读）
- `EffectResolver` = 运行时效果执行者（读取蓝图，操作 Combatant）

---

## 改动说明

### CardData.gd

移除 `damage` 和 `block` 字段，替换为：

```gdscript
class_name CardData
extends Resource

@export var card_name: String
@export var cost: int
@export var effects: Array[CardEffectData]
```

### EffectResolver.gd

改为遍历 `card.effects` 数组：

```gdscript
static func resolve(card: CardData, attacker: Combatant, defender: Combatant) -> void:
    for effect: CardEffectData in card.effects:
        if effect.type == "damage":
            defender.take_damage(effect.value)
        elif effect.type == "block":
            attacker.add_block(effect.value)
```

### CombatScreen.gd（仅 _rebuild_hand）

从 effects 数组提取值用于按钮文字显示：

```gdscript
func _rebuild_hand() -> void:
    for btn: Button in _hand_buttons:
        btn.queue_free()
    _hand_buttons.clear()
    for card: CardData in _engine.hand:
        var btn: Button = Button.new()
        var dmg: int = 0
        var blk: int = 0
        for effect: CardEffectData in card.effects:
            if effect.type == "damage":
                dmg = effect.value
            elif effect.type == "block":
                blk = effect.value
        btn.text = "%s\n费用:%d  攻:%d  挡:%d" % [card.card_name, card.cost, dmg, blk]
        btn.pressed.connect(_on_card_pressed.bind(card))
        _hand_container.add_child(btn)
        _hand_buttons.append(btn)
```

### 卡牌 .tres 文件格式

新格式使用 sub_resource 内嵌 `CardEffectData` 实例，`load_steps` 从 2 变为 4（2 ext + 1 sub + 1 root）。

**strike.tres 示例：**

```
[gd_resource type="Resource" script_class="CardData" load_steps=4 format=3]

[ext_resource type="Script" uid="uid://13mprie5g221" path="res://CardData.gd" id="1_CardData"]
[ext_resource type="Script" uid="<CardEffectData UID>" path="res://CardEffectData.gd" id="2_CardEffectData"]

[sub_resource type="Resource" id="Effect_damage"]
script = ExtResource("2_CardEffectData")
type = "damage"
value = 6

[resource]
script = ExtResource("1_CardData")
card_name = "打击"
cost = 1
effects = [SubResource("Effect_damage")]
```

**defend.tres**（block = 5）和 **bash.tres**（damage = 8）格式相同，各含一条 sub_resource。
