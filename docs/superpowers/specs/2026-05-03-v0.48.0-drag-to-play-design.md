# v0.48.0 拖拽出牌 Design

**Goal:** 将手牌出牌交互从两步点击（点卡牌 → 点敌人）改为拖拽连线（按下卡牌 → 拖向目标 → 松手出牌），提升手机触摸操作体验。

**Architecture:** 新增全屏 `DragLayer`（Control，最顶层），HandArea 检测初始按下并移交控制权，DragLayer 接管拖拽输入、创建幽灵牌、用 `_draw()` 实时绘制虚线连接目标；CombatScreen 监听 DragLayer 信号触发现有出牌逻辑。

**Tech Stack:** Godot 4 GDScript，严格类型，Control 绝对坐标，Tween 动画，CanvasItem._draw() 虚线绘制。

## 状态机

```
IDLE → LIFTED → DRAGGING → PLAYING → IDLE
                         ↘ CANCELLED → IDLE
```

| 状态 | 触发条件 | 行为 |
|------|----------|------|
| IDLE | 默认 | DragLayer mouse_filter=IGNORE |
| LIFTED | button_down 且能量充足 | 原卡牌 position.y -= 20，DragLayer 创建幽灵牌，mouse_filter=STOP |
| DRAGGING | 手指移动超过 8px | 幽灵牌跟随手指，连线出现，每帧检测目标 |
| PLAYING | 松手 + Y < 屏幕65%（SELF牌）或 松手 + Y < 屏幕65% + 有锁定目标（SINGLE/ALL牌） | emit card_played，清理幽灵牌，重置状态 |
| CANCELLED | 松手回到手牌区 或（非SELF牌且无有效目标） | 幽灵牌 Tween 飞回原位，原卡牌归位 |

能量不足时不进入 LIFTED，卡牌执行震动动画（±4px，0.1s × 3）作为拒绝反馈。

## 连线绘制

DragLayer 实现 `_draw()`，每帧由 `queue_redraw()` 刷新：

```gdscript
func _draw() -> void:
    if _state != State.DRAGGING:
        return
    var from := _ghost_card.position + Vector2(55, 0)  # 幽灵牌顶部中心
    for target_pos in _target_line_ends:
        draw_dashed_line(from, target_pos, Color(1, 1, 1, 0.85), 2.0, 12.0, 6.0)
```

- 普通牌（SINGLE）：`_target_line_ends` 含 1 个点，随左右拖动切换为最近敌人
- AoE 牌（ALL）：`_target_line_ends` 含所有存活敌人的点，固定不变
- 自身牌（SELF）：`_target_line_ends` 为空，不绘制连线，拖到手牌区以上松手即出牌

## 目标检测

```gdscript
func _detect_target(drag_x: float) -> int:
    var best := -1
    var best_dist := INF
    for i in _enemy_positions.size():
        var dist := abs(_enemy_positions[i].x - drag_x)
        if dist < best_dist:
            best_dist = dist
            best = i
    return best  # 始终锁定 X 最近的敌人，无死区
```

当前锁定目标的敌人 Panel 加高亮 `modulate = Color(1.3, 1.3, 1.0)`，切换时恢复前一个。DragLayer 通过信号通知 CombatScreen 更新高亮。

## 幽灵牌

- 类型：Panel + Label（不用 Button）
- 内容：与原卡牌相同文字
- 样式：`modulate.a = 0.9`，StyleBoxFlat 加 2px 白色边框
- 初始位置：原卡牌全局位置转换至 DragLayer 本地坐标
- 大小：`Vector2(110, 150)`（与手牌卡牌相同）

## 动画

**抬起**：即时（position.y -= 20，无 Tween，触摸须 ≤1 帧响应）

**取消归位**：
```gdscript
var tw := create_tween()
tw.tween_property(_ghost_card, "position", _origin_pos, 0.15).set_ease(Tween.EASE_OUT)
tw.tween_callback(_finish_cancel)
```

**出牌**：幽灵牌直接 `queue_free()`，`_rebuild_hand()` 重建手牌区。

**能量拒绝**：
```gdscript
var tw := create_tween()
tw.tween_property(btn, "position:x", btn.position.x + 4, 0.05)
tw.tween_property(btn, "position:x", btn.position.x - 4, 0.05)
tw.tween_property(btn, "position:x", btn.position.x + 4, 0.05)
tw.tween_property(btn, "position:x", btn.position.x, 0.05)
```

## CombatScreen 侧改动

1. `@onready var _drag_layer: DragLayer = $DragLayer`
2. `_ready()` 中连接信号：
   ```gdscript
   _drag_layer.card_played.connect(_on_drag_card_played)
   _drag_layer.drag_cancelled.connect(_on_drag_cancelled)
   ```
3. 手牌按钮改监听 `button_down`（替代 `pressed`）：
   ```gdscript
   btn.button_down.connect(_on_card_button_down.bind(i))
   ```
4. `_on_card_button_down(index)` → 检查能量 → 调用 `_drag_layer.begin_drag(...)`
5. `_on_drag_card_played(card_index, target_index)` → 调用现有 `_play_card(card_index, target_index)` 路径
6. 移除旧的 `_on_card_pressed()` 两步逻辑（`_pending_card_index`、`_set_targeting_mode()` 相关代码）
