# v0.20.0 架构清理 设计文档

**日期：** 2026-04-30
**目标：** 修复 v0.19.0 审计中发现的三处架构技术债，使代码符合 CLAUDE.md 四条架构原则。

---

## 核心决策

| 问题 | 决策 |
|---|---|
| NodeData extends Resource 但含运行时字段 | 方案 B：拆分为 NodeConfig（Resource）+ NodeData（RefCounted） |
| go_to_rest() 含治疗业务逻辑 | 移入 PlayerState.apply_rest_heal() |
| CombatScreen 直接写 player_state.hp | 新增 GameManager.end_combat(final_hp)，UI 只调接口 |

---

## 变更一：NodeConfig + NodeData 拆分

### 新增 NodeConfig.gd（Resource）

静态节点数据，可序列化为 .tres 文件：

```gdscript
class_name NodeConfig
extends Resource

enum Type { COMBAT, REST }
var type: Type = Type.COMBAT
var enemy_data: EnemyData
var column: int = 0
var row: int = 0
```

### 新增 data/nodes/ 目录，5 个 .tres 文件

| 文件 | type | enemy_data | column | row |
|---|---|---|---|---|
| node_a1.tres | COMBAT | jaw_worm.tres | 0 | 0 |
| node_a2.tres | COMBAT | fire_lizard.tres | 0 | 1 |
| node_b1.tres | REST | （空） | 1 | 0 |
| node_b2.tres | COMBAT | fire_lizard.tres | 1 | 1 |
| node_c1.tres | COMBAT | boss.tres | 2 | 0 |

### 修改 NodeData.gd（RefCounted）

只保留运行时图结构：

```gdscript
class_name NodeData
extends RefCounted

var config: NodeConfig
var connections: Array[NodeData] = []
```

### 影响

- `GameManager.start_new_run()`：preload 5 个 NodeConfig，构建 NodeData 包装，连线逻辑不变
- `GameManager.select_node()`：`node.type` → `node.config.type`
- `GameManager.get_current_enemy_data()`：`current_node.enemy_data` → `current_node.config.enemy_data`
- `MapScreen.gd`：`nd.type/column/row` → `nd.config.type/column/row`

## 变更二：治疗逻辑移入 PlayerState

### 修改 PlayerState.gd

新增字段和方法：

```gdscript
var last_rest_heal: int = 0

func apply_rest_heal() -> int:
    last_rest_heal = int(max_hp * 0.3)
    hp = mini(hp + last_rest_heal, max_hp)
    return last_rest_heal
```

### 修改 GameManager.go_to_rest()

```gdscript
func go_to_rest() -> void:
    player_state.apply_rest_heal()
    current_phase = Phase.REST
    get_tree().change_scene_to_file("res://rest/RestScreen.tscn")
```

删除 `GameManager.last_rest_heal` 字段。

### 修改 RestScreen.gd

```gdscript
_lbl_heal.text = "恢复了 %d 点生命" % GameManager.player_state.last_rest_heal
```

## 变更三：CombatScreen 不再直接写 player_state

### 新增 GameManager.end_combat(final_hp: int)

```gdscript
func end_combat(final_hp: int) -> void:
    player_state.hp = final_hp
    if is_final_node():
        go_to_win()
    else:
        go_to_reward()
```

### 修改 CombatScreen.gd

`_ready()` 中：
```gdscript
_btn_get_reward.pressed.connect(_on_proceed)
_btn_win.pressed.connect(_on_proceed)
```

新增：
```gdscript
func _on_proceed() -> void:
    GameManager.end_combat(_engine.player.hp)
```

`_on_combat_ended()` 中删除 `GameManager.player_state.hp = _engine.player.hp`，按钮可见性逻辑不变。
