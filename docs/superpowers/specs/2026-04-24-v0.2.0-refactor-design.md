# v0.2.0 架构重构设计文档

**日期**：2026-04-24  
**项目**：AIGameTest2（Godot 4.6 + GDScript）  
**范围**：v0.2.0 — 逻辑层与 UI 层分离（纯重构，零功能变化）

---

## 背景

v0.1.0 通过了全部验收标准，战斗循环可正常运行。但所有逻辑（回合控制、效果计算、UI 渲染）集中在 `CombatScreen.gd` 一个文件中。后续要加入地图、奖励、状态效果等功能时，这个单文件架构将快速失控。

v0.2.0 目标：在**不改变任何游戏行为**的前提下，将逻辑层从 UI 层剥离，为后续功能扩展打好基础。

---

## 目标文件结构

```
res://
├── CardData.gd          ← 不动
├── Combatant.gd         ← 不动
├── CombatEngine.gd      ← 新增：回合逻辑 + 游戏状态
├── EffectResolver.gd    ← 新增：卡牌效果计算
└── combat/
    ├── CombatScreen.tscn  ← 不动
    └── CombatScreen.gd    ← 改造：只负责 UI
```

---

## 职责划分

| 文件 | 职责 |
|------|------|
| `CombatEngine` | 持有游戏状态（玩家、敌人、牌堆、能量），管理回合流程 |
| `EffectResolver` | 纯函数：接收卡牌 + 双方角色，执行效果，无状态 |
| `CombatScreen` | 持有 Engine 引用，监听信号刷新 UI，把按钮事件转发给 Engine |

## CombatEngine 设计

**类定义：** `class_name CombatEngine extends RefCounted`

### 对外暴露的状态（只读）

```gdscript
var player: Combatant
var enemy: Combatant
var hand: Array[CardData]
var energy: int
var turn_number: int
```

### 对外方法

```gdscript
func setup() -> void
# 初始化玩家（70 HP）、敌人颚虫（44 HP）、硬编码牌库
# 内部调用 _start_player_turn()

func play_card(card: CardData) -> void
# energy < card.cost 时直接 return
# 调用 EffectResolver.resolve()，更新牌堆状态
# 发出 state_changed；若战斗结束则发出 combat_ended

func end_turn() -> void
# 弃掉手牌 → 调用 _do_enemy_turn() → 发出 state_changed
# 若战斗未结束，调用 _start_player_turn()
```

### 信号

```gdscript
signal state_changed
# 任何状态变化后发出，CombatScreen 统一调用 _refresh_ui()

signal combat_ended(result: String)
# result 为 "victory" 或 "game_over"
```

### 内部私有方法（从 CombatScreen 迁移）

- `_start_player_turn()` — 回合初始化：energy=3、player.block=0、抽牌、发出 state_changed
- `_draw_hand()` — 从牌库抽 5 张，牌库不足时洗入弃牌堆
- `_do_enemy_turn()` — 奇数回合攻击 11，偶数回合格挡 6
- `_check_end() -> bool` — 检查胜负，是则发出 combat_ended 并返回 true

## EffectResolver 设计

**类定义：** `class_name EffectResolver extends RefCounted`

### 唯一方法

```gdscript
static func resolve(card: CardData, attacker: Combatant, defender: Combatant) -> void
```

- `card.damage > 0` → `defender.take_damage(card.damage)`
- `card.block > 0` → `attacker.add_block(card.block)`

v0.2.0 只处理 damage 和 block。后续加脆弱、中毒等状态效果时，只需扩展此函数，CombatEngine 不用修改。

**约束：** EffectResolver 不持有任何状态，不引用 CombatEngine，可独立调用和测试。

## CombatScreen 改造后结构

改造后 CombatScreen 只做三件事：创建引擎、监听信号刷新 UI、把按钮事件转发给引擎。

```gdscript
var _engine: CombatEngine

func _ready() -> void:
    _engine = CombatEngine.new()
    _engine.state_changed.connect(_refresh_ui)
    _engine.combat_ended.connect(_on_combat_ended)
    _btn_end_turn.pressed.connect(_engine.end_turn)
    _engine.setup()

func _on_card_pressed(card: CardData) -> void:
    _engine.play_card(card)

func _refresh_ui() -> void:
    # 读 _engine.player / _engine.enemy / _engine.hand / _engine.energy / _engine.turn_number
    # 重建手牌按钮（逻辑与 v0.1.0 相同）

func _on_combat_ended(result: String) -> void:
    _lbl_result.text = "胜利！" if result == "victory" else "游戏结束"
    _lbl_result.visible = true
    _btn_end_turn.disabled = true
    for btn: Button in _hand_buttons:
        btn.disabled = true
```

预计代码量从 165 行缩减至 ~60 行。所有游戏逻辑从 CombatScreen 移除。

## 通信流程

```
玩家点击卡牌按钮
  → CombatScreen._on_card_pressed(card)
  → CombatEngine.play_card(card)
      → EffectResolver.resolve(card, player, enemy)
      → emit state_changed  （或 combat_ended）
  → CombatScreen._refresh_ui()

玩家点击 End Turn
  → CombatEngine.end_turn()
      → _do_enemy_turn()
      → emit state_changed
      → _start_player_turn()
      → emit state_changed
  → CombatScreen._refresh_ui()（触发两次，均安全）
```
