## 一、多境界架构

### 1.1 PlayerState

新增字段：
```gdscript
var current_realm: int = 0  # 0=筑基, 1=金丹, 2=元婴(未实装), 3=化神(未实装)
```

### 1.2 MapGenerator

- `realm == 0`：使用现有 `EARLY_GROUP_PATHS` / `MID_GROUP_PATHS` / `ELITE_GROUP_PATHS` / `BOSS_GROUP_PATH`

### 1.3 GameManager

新增方法 `advance_realm()`：
```gdscript
func advance_realm() -> void:
    player_state.current_realm += 1
    player_state.map_all_nodes.clear()
    player_state.available_nodes.clear()
    player_state.completed_nodes.clear()
    player_state.current_node = null
    pending_card_reward = true
    _setup_map()
```

`_setup_map()` 修改：
```gdscript
func _setup_map() -> void:
    var nodes: Array[NodeData] = MapGenerator.generate(player_state.current_realm)
    ...  # 其余不变
```

### 1.4 RewardScreen 境界推进触发

在 `RewardScreen` 结算完 Boss 奖励后（现有 `go_to_map` 调用处），当 `is_final_node() == true`：
- `current_realm < 1`（打完筑基）→ `GameManager.advance_realm()`
- `current_realm >= 1`（打完金丹，本版最终境界）→ `GameManager.go_to_win()`

`GameManager` 新增辅助方法：
```gdscript
func is_last_realm() -> bool:
    return player_state.current_realm >= 1  # v0.72.0 只有2境界
```

## 二、诅咒卡扩展

### 2.1 CardData 新增字段

```gdscript
@export var is_ye_huo: bool = false    # 业火：战斗开始时扣 HP
@export var is_bao_nu: bool = false    # 暴怒：打出任意牌时扣 HP
@export var is_kong_ju: bool = false   # 恐惧：封锁身法牌
@export var is_bei_shang: bool = false # 悲伤：回合结束弃手牌
```

### 2.2 各诅咒卡行为

**业火（`is_ye_huo`）**
- 触发：战斗开始（`turn_number == 1` 时，在 `_start_player_turn` 中）
- 扫描 `_deck + _discard_pile + hand` 中所有 `is_ye_huo` 的牌
- 每张扣 2 HP，不受格挡保护（直接 `player.hp -= count * 2`）

**暴怒（`is_bao_nu`）**
- 触发：每次 `play_card()` 时
- 统计当前手中（排除本次打出的牌）`is_bao_nu` 的数量 `n`
- 对玩家造成 `n * 2` 点伤害（不受格挡保护）

**恐惧（`is_kong_ju`）**
- 触发：`play_card()` 开始时检查
- 若手中有任意 `is_kong_ju` 且打出的牌 `card_type == "身法"` → 直接 return，阻止出牌
- CombatEngine 新增方法 `can_play_card(card: CardData) -> bool`：供 UI 层查询
- CombatScreen：若 `is_kong_ju` 在手中，身法牌显示为灰色（不可拖拽）

**悲伤（`is_bei_shang`）**
- 触发：`end_turn()` 中，处理完走火入魔之后
- 统计手中 `is_bei_shang` 数量 `n`
- 对每张：从手中随机选 1 张非诅咒牌（`not is_curse and not is_zahuorumuo and not is_venom and not is_ye_huo and not is_bao_nu and not is_kong_ju and not is_bei_shang`）移到 `_discard_pile`（弃置，非消耗）
- 若手中无非诅咒牌，跳过

### 2.3 .tres 文件

更新 `data/cards/ye_huo.tres`：加 `is_ye_huo = true`（保留 `is_curse = true`）

## 三、金丹境怪物

### 3.1 EnemyData 新增字段

```gdscript
@export var passive_block_per_turn: int = 0          # 每回合开始自动获得格挡
@export var emotion_card_damage_bonus: bool = false   # 攻击时加情绪牌数量伤害
@export var death_kills_others: bool = false          # 死亡时清除所有其他敌人
@export var skip_wave_gap_threshold: float = 0.0      # HP低于此比例时跳过wave_gap
@export var gains_strength_from_aoe: bool = false     # 被aoe_all命中后获得2力量
```

### 3.2 Combatant 新增字段

```gdscript
var draw_reduction: int = 0        # 下回合少抽N张，用完归零
var next_block_halved: bool = false # 下回合开始时格挡减半，用完清除
```

### 3.3 新增 Action Types

| 类型 | value | count | 说明 |
|------|-------|-------|------|
| `attack_self_damage` | 对玩家伤害 | 自损量 | 攻击玩家的同时对自身造成伤害 |
| `draw_penalty` | — | 减少量 | 减少玩家下回合抽牌数（count 存入 player.draw_reduction）|
| `attack_half_next_block` | 伤害 | — | 攻击+标记玩家 next_block_halved=true |
| `block_all_enemies` | 格挡量 | — | 给场上所有存活敌人获得格挡 |
| `aoe_all` | 伤害 | — | 对玩家+所有敌人造成伤害（敌人不受格挡影响）|
| `attack_bao_nu` | 伤害 | 数量 | 攻击+往弃牌堆塞 count 张暴怒 |
| `attack_kong_ju` | 伤害 | 数量 | 攻击+往弃牌堆塞 count 张恐惧 |
| `attack_bei_shang` | 伤害 | 数量 | 攻击+往弃牌堆塞 count 张悲伤 |
| `wave_gap` | — | — | 无攻击无格挡；HP < skip_wave_gap_threshold 时被跳过 |

### 3.4 CombatEngine 新增逻辑

**passive_block_per_turn：** 在 `_do_enemy_turn()` 内、执行 `enemies[i].block = 0` 之前，若 `data.passive_block_per_turn > 0`，则先 `enemies[i].add_block(data.passive_block_per_turn)`，再清零。这样每回合敌人行动前先获得格挡，然后格挡生效抵挡当轮任何反击。

**aoe_all：** 先对玩家造成 value 伤害（调用 `_enemy_attack` 但不受格挡），再遍历所有存活敌人造成 value 伤害（直接 `enemies[j].hp -= value`）。若敌人有 `gains_strength_from_aoe == true`，伤害后 `enemies[i].strength += 2`。

**death_kills_others：** 在 `_check_end()` 中，每当一个敌人 HP 归零时，若其 data 有 `death_kills_others == true`，遍历所有其他敌人将 HP 设为 0。

**skip_wave_gap：** 在 `_resolve_enemy_actions()` 中，确定下一个 action 后，若 `action.type == "wave_gap"` 且 `float(enemies[i].hp) / data.hp < data.skip_wave_gap_threshold`，则将 action_index 推进 1 并取下一个 action。

**draw_reduction：** 在 `draw_cards(n)` 内，实际抽牌数 = `max(0, n - player.draw_reduction)`，然后 `player.draw_reduction = 0`。

**next_block_halved：** 在 `_start_player_turn()` 中，如果 `player.next_block_halved == true`，则 `player.block = player.block / 2`，然后 `player.next_block_halved = false`。
