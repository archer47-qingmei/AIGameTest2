# v0.35.0 剑修奖励牌池设计

**目标：** 将战斗奖励的选牌池替换为剑修专属卡牌；奖励池按职业区分，为后续法修/体修预留扩展位置。

---

## 数据层

### CardPool.gd

新增常量 `SWORD_REWARD_CARDS`，包含全部 6 张剑修卡：

```gdscript
const SWORD_REWARD_CARDS: Array[String] = [
    "res://data/cards/zhan_tie.tres",
    "res://data/cards/po_feng.tres",
    "res://data/cards/ce_bu.tres",
    "res://data/cards/lian_xi.tres",
    "res://data/cards/chuan_xin.tres",
    "res://data/cards/yang_jian_shu.tres",
]
```

原有 `CARDS` 常量保持不动，作为未匹配职业时的兜底池。

---

## 逻辑层

### RewardEngine.gd

`get_options()` 签名改为接受 `character: String` 参数：

```gdscript
static func get_options(character: String) -> Array[CardData]:
    var pool_paths: Array[String] = []
    match character:
        "sword":
            pool_paths.assign(CardPool.SWORD_REWARD_CARDS)
        _:
            pool_paths.assign(CardPool.CARDS)
    var pool: Array[CardData] = []
    for path in pool_paths:
        pool.append(load(path) as CardData)
    pool.shuffle()
    return pool.slice(0, mini(3, pool.size()))
```

shuffle + slice(0, 3) 逻辑不变，只替换牌池来源。

调用处改为传入职业参数：

RewardEngine.get_options(GameManager.player_state.character)

## 不在本版范围内

- 灵品/仙品剑修卡（高阶奖励牌）——后续版本添加到 `SWORD_REWARD_CARDS`
- 法修/体修的奖励池——职业解锁时同步添加对应 `FA_REWARD_CARDS` / `TI_REWARD_CARDS`
- 奖励池去重（已有牌不再出现）——当前池只有 6 张，重复获取是正常设计
