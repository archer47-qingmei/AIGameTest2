# v0.27.0 十关卡滚动地图 Design Spec

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将地图从 4 列扩展到 10 列，新增可滚动地图界面，适配 480×854 竖屏，玩家从底部起点向上滑动至 Boss。

**Architecture:** 重构 `MapGenerator.gd` 为通用列生成逻辑（按 phase 分配节点类型），修改 `MapScreen.gd` 坐标映射公式，并在 `MapScreen.tscn` 中加入 ScrollContainer 层级。其余模块（RewardEngine、ShopEngine、NodeConfig、连线拓扑）不变。

**Tech Stack:** Godot 4 GDScript，项目分辨率 480×854（已有）。

---

## Section 1: MapGenerator 重构

将 `_make_col0/1/2/3()` 四个硬编码方法替换为通用 `_make_column(col: int)` 方法，新增常量 `TOTAL_COLUMNS = 10`。

节点类型分配规则：

| Phase | 列范围 | row 0 | row 1 |
|-------|--------|-------|-------|
| 早期 | col 0–3 | COMBAT | COMBAT |
| 中期 | col 4–5 | COMBAT | REST |
| 后期 | col 6–7 | ELITE | COMBAT |
| 精英期 | col 8 | ELITE | ELITE |
| Boss | col 9 | BOSS（居中，无 row） | — |

**商店规则：** 在 col 4–8 中随机选 2 列，将该列的 COMBAT 节点（不覆盖 ELITE/REST）改为 SHOP。全局恰好 2 家商店，位置每局随机。

**奖励赋值：** 精英节点（ELITE）赋 `burning_gem`，Boss 节点赋 `life_ring`，逻辑与现有一致，迁移到 `_make_column()` 中。

`generate()` 对外接口不变，返回 `Array[NodeData]`。

## Section 2: MapScreen 滚动布局

### 场景结构

```
MapScreen (Control, 480×854)
  └─ ScrollContainer (480×854, scroll_vertical=true)
       └─ MapContent (Control, 480×1500)
```

`MapContent` 高度 = `(TOTAL_COLUMNS - 1) × 150 + 300 = 1500px`。

### 坐标映射

`_get_node_pos()` 新公式（坐标相对于 MapContent）：

- Boss（col 9）：`Vector2(240.0, 150.0)`
- 其余列：`col_y = 1350.0 - col * 150.0`，`row_x = 140.0 if row == 0 else 340.0`

| col | y |
|-----|---|
| 0（起点）| 1350 |
| 1 | 1200 |
| 2 | 1050 |
| 3 | 900 |
| 4 | 750 |
| 5 | 600 |
| 6 | 450 |
| 7 | 300 |
| 8 | 150+150=300 → 实为 1350-8×150=150 |
| 9（Boss）| 150（单独处理）|

初始化时将 ScrollContainer 滚动到底部（`scroll_vertical = scroll_vertical_max`），玩家从起点向上滑动。

### 不需要修改的部分

- `_draw()`（连线绘制）：坐标由 `_get_node_pos()` 提供，换到 MapContent 坐标系后自动正确
- 按钮生成逻辑：同上
- 节点连线拓扑：基于 `column` 字段，10 列可直接复用
