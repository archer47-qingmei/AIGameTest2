# v0.22.0 奖励界面重构 设计文档

**日期：** 2026-04-30
**版本：** v0.22.0
**主题：** 奖励界面两层结构 + 跳过卡牌 + 遗物作为可选战利品

---

## 目标

重构奖励界面，使其符合以下设计：
1. 战利品列表层（第一层）：按战斗类型展示可领取的奖励项
2. 卡牌选择层（第二层）：点击卡牌项后进入，可选一张或跳过
3. 遗物直接在列表层展示并点击领取，无需进入子界面
4. 常驻「继续」按钮，随时可离开奖励界面

---

## 游戏流程

```
普通战斗胜利
  → 奖励界面（战利品列表）
      [卡牌]          ← 点击进入卡牌选择
      [继续]          ← 随时可点

精英/Boss 战斗胜利
  → 奖励界面（战利品列表）
      [遗物名称 — 描述]  ← 点击直接领取，按钮变灰
      [卡牌]             ← 点击进入卡牌选择
      [继续]             ← 随时可点

卡牌选择子界面
  → 显示 3 张卡牌 + [跳过]
  → 选一张或跳过 → 返回战利品列表，[卡牌] 按钮变灰
```

## 架构变化

### GameManager.gd

新增字段：
```gdscript
var pending_relic: RelicData = null
```

修改 `end_combat()`：
- 精英战：设置 `pending_relic = burning_gem`，调用 `go_to_reward()`
- Boss 战：设置 `pending_relic = life_ring`，调用 `go_to_reward()`（不再直接 `go_to_win()`）
- 普通战：`pending_relic = null`，调用 `go_to_reward()`
- 移除原有的 `_grant_relic()` 自动发放逻辑（改为玩家主动领取）

`go_to_reward()` 不变。

### RewardScreen.gd

重构为两个内部状态：

**LIST 状态（战利品列表）**
- 若 `GameManager.pending_relic != null`：显示遗物按钮（文本：「[名称] — [描述]」）
  - 点击 → 将遗物 duplicate 追加到 `player_state.relics`，清空 `pending_relic`，按钮禁用
- 显示卡牌按钮（文本：「卡牌」）
  - 点击 → 切换至 CARD_PICK 状态
  - 已选过卡牌后：按钮禁用（文本变为「卡牌 ✓」）
- 常驻「继续」按钮
  - 点击 → 根据 `GameManager.is_final_node()` 跳转 `go_to_win()` 或 `go_to_map()`

**CARD_PICK 状态（卡牌选择）**
- 隐藏战利品列表，显示 3 张卡牌按钮 + 「跳过」按钮
- 选一张卡牌 → 加入 `player_state.deck`，回到 LIST 状态，卡牌按钮禁用
- 点击「跳过」→ 不加牌，回到 LIST 状态，卡牌按钮禁用

### RewardEngine.gd

不需要修改（仍返回 3 张卡牌选项）。

## 验收标准

| 场景 | 预期行为 |
|---|---|
| 普通战斗胜利 | 奖励界面只显示「卡牌」按钮 + 「继续」按钮 |
| 精英战斗胜利 | 奖励界面显示遗物按钮 + 「卡牌」按钮 + 「继续」按钮 |
| 点击遗物按钮 | 遗物加入 `player_state.relics`，按钮变灰 |
| 点击「卡牌」 | 进入卡牌选择，显示 3 张卡 + 「跳过」|
| 选一张卡 | 卡加入牌组，返回列表，「卡牌」按钮变灰 |
| 点击「跳过」 | 不加牌，返回列表，「卡牌」按钮变灰 |
| 点击「继续」（非 Boss）| 跳转地图界面 |
| Boss 战胜利 + 点击「继续」| 跳转胜利界面 |
| 未领任何奖励直接点「继续」| 正常跳转，不报错 |
