# 心魔劫 Boss 实现设计文档 (v0.61.0)

## Goal

实现心魔劫 Boss：单体，HP=270，核心机制为复制玩家招式牌，包含心魔反噬、
自我崩解被动、第三阶段走火入魔压制，以及新诅咒牌走火入魔。

## Architecture

心魔劫的动态行为通过三处扩展实现：
1. EnemyData 新增三个字段控制复制和周期行动
2. CombatEngine 在 setup 时扫描牌组、存入复制卡池，在 resolve/execute
   阶段处理新动作类型和被动
3. CardData 新增 is_zahuorumuo 标记，end_turn 处理其弃牌效果

## 阶段结构

| 阶段 | 触发 | 行动 |
|------|------|------|
| 第一阶段 | HP ≥ 50% | mirror_attack（随机复制招式） |
| 第二阶段 | HP < 50% | 同上 + 被动：每回合自失3HP、获得3力量 |
| 第三阶段 | HP < 30% | attack_zahuorumuo(12, count=1) |

心魔反噬（xin_mo_fanshi）每3回合覆盖当前阶段的常规行动，三阶段均触发。

## 新诅咒牌：走火入魔

- 无法打出
- 被摸到时留在手中（不自动弃牌，与心魔不同）
- 回合结束时：随机弃掉手中另1张非诅咒牌；若无其他牌则跳过
- 走火入魔本身在回合结束后进入弃牌堆

CardData 新增 `is_zahuorumuo: bool = false`，不使用 is_curse（is_curse
的语义是"摸到即弃"，走火入魔不符合）。

## 复制招式牌（mirror_attack）

**复制规则：**
- setup() 时扫描 _draw_pile，筛选满足以下条件的 CardData：
  - card_type == "招式"
  - effects 数组中所有元素均为 "damage" 类型（即只有伤害效果，无副效果）
- 随机取最多5张存入 `_boss_copied_cards: Array[CardData]`
- 若无符合条件的招式牌，回退为固定值 value=8，display_label="攻击"

**预解析（_resolve_enemy_actions）：**
- mirror_attack 类型：随机从 _boss_copied_cards 选1张，写入
  action.value（伤害值）和 action.display_label（牌名）
- 执行时调用 _enemy_attack(enemies[i], action.value)，力量/易伤正常生效

**意图显示：** `"意图：%s %d" % [action.display_label, action.value]`，
例如"意图：裂空 10"

## 心魔反噬（xin_mo_fanshi）

**触发：** EnemyData.periodic_interval = 3，turn_number % 3 == 0 时在
_resolve_enemy_actions 中覆盖当前动作。

**执行：**
1. 筛出手牌中所有非诅咒牌（not is_curse and not is_zahuorumuo）
2. 若存在：随机选1张，从手牌移除，插入 CURSE_CARD.duplicate()
3. 若不存在：跳过

**意图：** `"意图：心魔反噬"`

## 第二阶段被动（HP < 50%）

在 _do_enemy_turn 处理完当前动作后，检查：
- `float(enemies[i].hp) / float(enemies[i].max_hp) < 0.5`
- `_enemy_data_list[i].copies_player_cards == true`

若满足：`enemies[i].hp = max(0, enemies[i].hp - 3)`，
`enemies[i].strength += 3`。HP 归零时正常触发胜利判定。

## 第三阶段（attack_zahuorumuo）

type="attack_zahuorumuo"，value=12，count=1。

执行：`_enemy_attack(enemies[i], 12)` + 向 _discard_pile 追加
count 张走火入魔（zao_huo_ru_mo.tres）的 duplicate()。
