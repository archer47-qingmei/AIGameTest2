# v0.14.0 休息节点 设计文档

**日期：** 2026-04-29
**目标：** 地图扩展为 4 节点，在 Boss 前插入固定休息节点，玩家可回复最大 HP 的 30%。

---

## 核心决策

| 问题 | 决策 |
|---|---|
| 节点数量 | 4（普通 → 普通 → 休息 → Boss） |
| 休息节点表示方式 | `enemy_sequence` 中插入字符串 `"rest"`（魔法字符串） |
| 回血量 | `int(max_hp * 0.3)` = 21 点，上限不超过 max_hp |
| 休息场景 | 新建独立场景 `rest/RestScreen.tscn` |

---

## 架构总览

```
逻辑层
  GameManager.gd     序列生成加入 "rest"；新增 go_to_rest()；Phase 枚举新增 REST
  PlayerState.gd     新增 max_hp: int = 70

UI 层
  map/MapScreen.gd          区分节点类型，休息节点显示"休息站"，点击路由
  rest/RestScreen.tscn（新） 标题 + 回血信息 + 当前 HP + 继续按钮
  rest/RestScreen.gd（新）  _ready() 回血并更新显示，按钮跳地图
```

**数据流：**

```
start_new_run()
  → enemy_sequence = [普通1, 普通2, "rest", boss]

MapScreen 显示 4 个节点
  → 节点 2 标为"休息站"，节点 3 标为"关卡 3（Boss）"
  → 点击休息节点 → GameManager.go_to_rest()
  → 点击战斗节点 → GameManager.go_to_combat()

RestScreen._ready()
  → heal = int(player_state.max_hp * 0.3)
  → player_state.hp = min(player_state.hp + heal, player_state.max_hp)
  → 显示回血量和当前 HP
  → 点击"继续" → GameManager.go_to_map()（current_node + 1）
```

## 逻辑层

### GameManager.gd

**Phase 枚举新增 REST：**
```gdscript
enum Phase { MENU, MAP, COMBAT, REWARD, REST, WIN }
```

**`start_new_run()` 序列生成：**
```gdscript
var pool: Array[String] = NORMAL_POOL.duplicate()
pool.shuffle()
player_state.enemy_sequence.assign(pool)
player_state.enemy_sequence.append("rest")
player_state.enemy_sequence.append(BOSS_ENEMY)
```

**新增 `go_to_rest()`：**
```gdscript
func go_to_rest() -> void:
    current_phase = Phase.REST
    get_tree().change_scene_to_file("res://rest/RestScreen.tscn")
```

`get_current_enemy_data()` 和 `is_final_node()` 不需要改动。

### PlayerState.gd

新增字段：
```gdscript
var max_hp: int = 70
```

## UI 层

### MapScreen.gd

节点标签计算逻辑改为独立方法：
```gdscript
func _get_node_label(i: int) -> String:
    var path: String = GameManager.player_state.enemy_sequence[i]
    if path == "rest":
        return "休息站"
    elif i == GameManager.player_state.enemy_sequence.size() - 1:
        return "关卡 %d（Boss）" % (i + 1)
    else:
        return "关卡 %d" % (i + 1)
```

点击路由：
```gdscript
func _on_node_pressed(i: int) -> void:
    GameManager.player_state.current_node = i
    var path: String = GameManager.player_state.enemy_sequence[i]
    if path == "rest":
        GameManager.go_to_rest()
    else:
        GameManager.go_to_combat()
```

### RestScreen.tscn（新）

```
RestScreen (Control, fullscreen)
  └── VBoxContainer
        ├── LblTitle    ("休息站")
        ├── LblHeal     ("恢复了 XX 点生命")
        ├── LblHP       ("当前生命：XX / 70")
        └── BtnContinue ("继续")
```

### RestScreen.gd（新）

```gdscript
extends Control

@onready var _lbl_heal: Label      = $VBoxContainer/LblHeal
@onready var _lbl_hp: Label        = $VBoxContainer/LblHP
@onready var _btn_continue: Button = $VBoxContainer/BtnContinue

func _ready() -> void:
    var state: PlayerState = GameManager.player_state
    var heal: int = int(state.max_hp * 0.3)
    state.hp = min(state.hp + heal, state.max_hp)
    _lbl_heal.text = "恢复了 %d 点生命" % heal
    _lbl_hp.text = "当前生命：%d / %d" % [state.hp, state.max_hp]
    _btn_continue.pressed.connect(GameManager.go_to_map)
```

## 技术债

| 编号 | 描述 | 影响 |
|---|---|---|
| TD-001 | `max_hp = 70` 分别硬编码在 `CombatEngine.gd`（`player.max_hp = 70`）和 `PlayerState.gd`（`max_hp: int = 70`）两处。后续如果要支持角色选择或 HP 升级，需统一到单一来源。 | 低——两处值相同，当前不产生矛盾；但修改时需同步两处 |
| TD-002 | `enemy_sequence` 混存敌人路径和 `"rest"` 字符串，类型语义不纯。后续扩展精英节点或商店节点时，建议升级为 `NodeData` resource 方案。 | 低——当前只有两种类型，`if path == "rest"` 判断简单明确 |
